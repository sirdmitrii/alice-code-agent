# Alice Code

Консольный кодовый агент (аналог Claude Code), работающий на **Алисе от Яндекса**.
Умеет читать/писать/править файлы и запускать команды в рабочей папке — в режиме
диалога, как Claude Code, но на бесплатной веб-Алисе.

> ⚠️ Это работает поверх **веб-сессии Алисы** (реверс websocket-протокола uniproxy),
> а не официального API. Для личного использования — ок; это неофициально, хрупко
> (Яндекс может поменять протокол) и формально против ToS. Нужен аккаунт Яндекса.

## Что нужно

- **Windows** 10/11
- **Python 3.10+** — поставь с https://python.org и на установке отметь галочку
  «Add python.exe to PATH»
- Интернет (при первом запуске скачаются зависимости и браузер ~150–300 МБ)
- Аккаунт Яндекса (для режима Pro — желательно с Яндекс Плюс)

## Как запустить

1. Распакуй архив в любую папку.
2. Запусти **`run.bat`** двойным кликом.
3. При первом запуске он сам:
   - создаст виртуальное окружение и поставит зависимости;
   - скачает браузер для входа (один раз);
   - **откроет окно браузера — залогинься в Яндекс**. Окно закроется само.
4. Появится приглашение `›` — пиши задачи обычным текстом.

Логин сохраняется, повторно входить не нужно. Когда токен протухает — обновляется
сам (если надо — снова всплывёт окно входа).

## Команды в чате

- обычный текст — задача агенту («покажи файлы», «создай скрипт …», «почини баг в …»)
- многострочный текст можно **вставлять (Ctrl+V)** — отправка по Enter; история — стрелки ↑/↓
- **прикрепить файл (Ctrl+V):** скопируй файл в Проводнике и вставь путь, либо вставь
  скриншот из буфера — Алиса прочитает файл (картинки, PDF, docx, txt/код и др.).
  Файл обрабатывается пару секунд, ответ приходит следом
- `/resume [id]` — вернуться к прошлой сессии (без id — список для выбора)
- `/clear` — начать новую сессию
- `/trust [all|danger|none]` — уровень подтверждений: спрашивать на все операции /
  только на опасные (по умолчанию) / не спрашивать вовсе
- `/help` — помощь
- `/exit` — выход

При нескольких операциях в одном ответе подтверждение запрашивается **один раз
на все** (а не по отдельности).

Каждый запуск — новая сессия со своим диалогом (на сайте Алисы виден как один
диалог, а не множество). Сессии сохраняются в `.alice_sessions/` и доступны через
`/resume`.

## Что умеет

Инструменты: `read_file` (с номерами строк и диапазоном), `write_file`,
`edit_file` (с `replace_all`), `list_dir`, `glob` (поиск файлов по маске),
`grep` (поиск по содержимому), `run_command`.

**Память проекта:** если в рабочей папке есть `CLAUDE.md` / `AGENTS.md` / `.alice.md`,
агент подхватит их в системный промпт.

**Контекст:** длинная история автоматически сжимается в сводку (через Алису),
старые результаты инструментов сворачиваются — чтобы не упираться в лимит.

**Проверка после правок:** задай `ALICE_VERIFY_CMD` в `.env` (напр. `pytest -q`) —
после изменений файлов агент прогонит её и при ошибке попросит Алису исправить.
Опасные операции (запись/правка/команды) спрашивают подтверждение.
Все пути ограничены рабочей папкой.

Рабочую папку можно задать в `.env` (`PROJECT_DIR=`), по умолчанию — папка запуска.

## Если что-то не так

- **Python не найден** — поставь Python 3.10+ с галочкой «Add to PATH», перезапусти.
- **Ответы короткие/слабые (режим Base)** — значит вход не выполнен. Перезапусти
  `run.bat` и залогинься в окне браузера (нужен режим Pro).
- **Переустановить вход с нуля** — запусти `.venv\Scripts\python.exe alice_session.py login`.
- Логи адаптера — в `adapter.log`.
